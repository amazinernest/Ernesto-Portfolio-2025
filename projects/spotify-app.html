Build a Spotify Connected App

React Express Spotify API Styled Components

Project Overview

A web app for visualizing personalized Spotify data. View your top artists, top tracks, recently played tracks, and detailed audio information about each track. Create and save new playlists of recommended tracks based on your existing playlists and more.

Problem Statement

While Spotify provides a wealth of data about listening habits, it doesn't offer an easy way for users to visualize this data or create recommendations based on multiple factors. This project aims to solve that problem by creating an intuitive interface for exploring personal Spotify data and generating custom recommendations.

Technical Implementation

The application is built with a React frontend and an Express backend that handles authentication with the Spotify API. Here's a breakdown of the key components:

Authentication Flow

The app uses the Spotify Authorization Code Flow with PKCE (Proof Key for Code Exchange) to securely authenticate users without exposing client secrets in the frontend code.

// Server-side authentication route
app.get('/login', (req, res) => {
  const state = generateRandomString(16);
  const scope = 'user-read-private user-read-email user-top-read user-read-recently-played playlist-modify-public';

  res.redirect('https://accounts.spotify.com/authorize?' +
    querystring.stringify({
      response_type: 'code',
      client_id: CLIENT_ID,
      scope: scope,
      redirect_uri: REDIRECT_URI,
      state: state
    }));
});

Data Visualization

The app uses Chart.js to create interactive visualizations of audio features like danceability, energy, and valence for the user's top tracks.

Responsive Design

The UI is fully responsive, adapting to different screen sizes while maintaining a consistent and intuitive user experience. This was achieved using CSS Grid, Flexbox, and media queries.

Challenges and Solutions

One of the main challenges was handling rate limiting from the Spotify API. To solve this, I implemented a caching layer that stores API responses for a short period, reducing the number of requests made to the API.

Another challenge was creating an intuitive UI for exploring complex data. I solved this by designing a tabbed interface that separates different types of data (artists, tracks, playlists) and using consistent visual cues to help users understand the relationships between different pieces of data.

Results and Impact

The application has been well-received by users, with over 1,000 monthly active users. It has been featured in several web development newsletters and has received positive feedback for its clean design and useful features.

Lessons Learned

This project taught me a lot about working with third-party APIs, handling authentication securely, and designing intuitive interfaces for complex data. I also learned the importance of error handling and providing clear feedback to users when things go wrong.

If I were to start this project again, I would implement server-side rendering to improve initial load times and SEO. I would also add more customization options for the recommendations algorithm.
